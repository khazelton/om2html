Backend - a server backend¶

mod:‘backend’ – server backend
Author:Roland Hedberg
Version:1.1

Module¶

This module is expected to be used by applications that acts as JSON-RPS, REST, SOAP, .. servers.

All operations are atomic, that is: either they succeed in their entirety or they fail.

class backend2.Backend(om2, authzsrv=None, log=None, debug=0, verbose=0)¶

A interface to a OM2 cloud which is expected to be used as part of a AJAX/REST/SOA service

add(obj, req=None)¶

Add a object to the OM2 system. If no uriref are given one will be constructed on-the-fly.

Parameters:
  • obj – object instance as {object_type: [uriref, ava]} where ava is a dictionary.
  • req – The entity that wants this operation to be completed
Returns:

True or False

delete(uriref, req=None)¶

Delete an object from the OM2 system

Parameters:
  • uriref – The identifier of the object
  • req – The requestor, an identifier of the entity that wants to run this operation.
Returns:

True if the object was removed, or False is it for some reason couldn’t be removed. Removing a non-existent object should result in a True result.

read(uriref, properties=[], expand=False, req=None, native=False)¶

Read information about an object from the OM2 system

Parameters:
  • uriref – The identifier of the object
  • properties – A list of properties which should be returned if present in the object
  • req – The requestor, an identifier of the entity that wants to run this operation.
  • expand – Read objects that this object are referring to.
  • native – Return the objects in their native RdfMarshal format and not as a combination of dictionaries,lists and singel values as produced by the Base.struct() command.
Returns:

The object or None if the object doesn’t exist or if the requestor are not allowed to read the object.
search(objtype, prefilt={}, properties=[], req=None)¶

Search for objects in the OM2 system

Parameters:
  • objtype – The type of the object
  • prefilt – The original search filter as a dictionary
  • properties – A list of properties which should be returned if present in the found objects.
  • req – The requestor, an identifier of the entity that wants to run this operation.
Returns:

The objects or None if no objects was found or if the requestor was not allowed to see the objects found.
